----
説明

	ディレクトリ -> 圧縮 -> 暗号化 -> 単独のファイル
	と、その逆を行います。

----
コマンド

	> WCluster.exe [/F] [/P パスフレーズ] [/S] [/O デフォルト出力パス] [/-] 入力パス [出力パス]

	入力パスがディレクトリの場合、ファイル（クラスタファイル）を作成します。
	入力パスがファイル（クラスタファイル）の場合、ディレクトリに展開します。

	出力パスが存在するディレクトリの場合、そのディレクトリの直下にそれなりの名前で出力します。
	出力パスを省略すると、デフォルト出力パスを使用しますが、
	デフォルト出力パスも省略されていた場合、入力パスと同じ場所にそれなりの名前で出力します。

	/F を指定すると、出力パスを強制的に上書きします。

	/S を指定すると、暗号化・復号時コンソールを表示します。

	/- までのオプションは順不同です。/- を指定するとオプション終了と見なします。

	パスフレーズを指定すると、暗号化・復号を行います。
	復号に失敗するとエラーになります。

	非同期で実行するので、同期する場合は START /WAIT して下さい。

----
パスフレーズ

	制限

		空文字列ではないこと。
		空白以外のコントロールコードを含まないこと。
		Shift_JIS で表現できる文字列であること。

	パスフレーズの拡張

		パスフレーズの終わりを以下のようにすると、パスフレーズを拡張します。

			[CNN]

			C = 任意の半角文字
			N = 数字

			NN は 20 〜 50 の整数を指定して下さい。

		パスフレーズの終わりの [CNN] を削除して、C を 2 ^ NN 文字追加します。
		パスフレーズが非常に長くなるため、鍵生成に時間が掛かり、総当たり攻撃対策になるような気がします。

		あまり NN を大きくし過ぎるといつまでも処理が終わりません。25 〜 30 あたりからお試し下さい。

		例えば tiger1[x25] というパスフレーズは...

			tiger1xxxxxxxxxx ... xxxxxxxxxx
			      |<----- 2^25 文字 ----->|

		...になります。

----
実行例

	> WCluster.exe C:\Data\0001

	> START /WAIT WCluster.exe /F /P abc123[Z30] /S Input\abc Output\def
